# Engineer Onboarding — Your First Two Weeks

This page is the canonical onboarding checklist for new Northwind engineers.

## Week 1

- Get your laptop, hardware token, and access to the corporate VPN.
- Read `wiki://intro` if you haven't already.
- Pair with your onboarding buddy on a "first PR" — usually a doc fix or a small refactor.
- Attend the architecture overview deep-dive (recurring Wednesdays).
- Set up your local dev environment using `tools/bootstrap.sh`.

## Week 2

- Shadow on-call for two days — read every alert that fires, ask why.
- Pick a starter ticket from the "good first issue" board.
- Get added to the relevant team-channel oncall rotation.
- Schedule 30-minute intros with your three closest dependency-team leads.

## Tools you'll use daily

- VS Code with the Northwind workspace settings.
- The internal `nw` CLI for tenant-aware test data.
- Grafana for metrics, Loki for logs, Tempo for traces.
