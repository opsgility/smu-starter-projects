# Task Tracker App — GitHub Projects Starter

A sample application to practice GitHub Projects V2 for project management.

## Files

- `app.js` — Application with a feature backlog listed as comments

## Status

Under active development. See Issues for planned work.

## What You Will Do

In the exercises you will:
1. Push this project to a new private GitHub repository
2. Create 8 GitHub Issues from the feature backlog in `app.js`
3. Create a GitHub Project board with `gh project create`
4. Add all issues to the project with `gh project item-add`
5. Add custom fields: **Priority** (Critical/High/Medium/Low), **Effort** (Small/Medium/Large), and **Sprint** (iteration)
6. Update issue statuses to simulate a Kanban board (Todo / In Progress / Done)
7. Create milestones (v1.0 MVP, v1.1 Polish, v2.0 Collaboration) and assign issues
8. Explore Board, Table, and Roadmap views
9. Configure built-in automations (auto-add, auto-close on merge)
10. Close issues and observe milestone progress tracking

## GitHub Projects Views

| View | Best For |
|------|----------|
